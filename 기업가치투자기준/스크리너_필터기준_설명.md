# 스크리너 필터 기준 설명서 (`screener.py`)

> `screener.py`는 **나스닥_후보발굴_프롬프트.md** / **코스피_후보발굴_프롬프트.md** 기준을
> Yahoo Finance 실시간 데이터로 자동 계산해 매수 후보를 추려주는 도구입니다.
>
> **2026-07 전면 개편**
> - 구 듀얼모드(★눌림목 + ☆모멘텀) → **나스닥 3유형(눌림목 / 돌파 / 에너지응축) + 코스피 10점 모드**
> - 종목: 나스닥 **~220개**, 코스피 **~120개**
> - 조회: **병렬 12스레드** (나스닥 ~15초, 코스피 ~10초)

---

## 0. 사용법

```bash
python3 screener.py            # 나스닥 스캔 (기본)
python3 screener.py --kospi    # 코스피 스캔
```

개념 설명은 `1·2단계_쉽게_이해하기.md`를 함께 보세요.

## 1. 나스닥 모드 — 개요

### 흐름도

```
나스닥 후보 종목 (~220개)
      │
  ① 시장 레짐 (QQQ > 50MA + 50MA 우상향)
      → 약세여도 후보는 표시, 실제 진입은 보류 권장 (하드 차단 아님)
      │
  ② 기본 필터 (시총 ≥ $10B / 부채비율 ≤ 200%)
      │
  ②-b 펀더 필터 (ROE ≥ 7% · 매출성장 ≥ 5% · **forward PEG** ≤ 2.5 · **trailing PER** ≤ 80)
      → 기술 통과여도 미충족 시 진입유형 제외 (별도 섹션 표시)
      │
  ③ 추세 필터 (주가 > 50MA + 50MA 우상향)   → 미충족 시 진입유형 없음(추세X)
      │
  ④ QQQ 상대강도 (20일 > QQQ) — 진입 유형 **공통 필수**
      │
  ⑤ 진입 유형 분류 (우선순위: 눌림목 > 돌파 > 에너지응축)
      ├─ 눌림목   : 정배열 + 5MA ±2.5% + 20MA이격≤8% + RSI 38~65 + 거래량 수축
      ├─ 돌파     : 고점 -6% 이내 + RSI 58~82 + 거래량 급증
      └─ 에너지응축: MA 간격 좁음 + 20일 레인지 ≤14% + RSI 43~70
      │
  ⑤ QQQ 상대강도 Top 10 (20일 수익률 기준)
```

### 진입 유형 3가지

| 유형 | 설명 | 핵심 지표 |
|------|------|----------|
| **눌림목** | 상승추세 중 5/20MA 근처 조정 → 재진입 | 5MA이격 ±2.5% + 거래량 수축 |
| **돌파** | 52주 고점 근접 + 거래량 동반 돌파 시도 | 고점 -6% 이내 + vol 5d/20d ≥ 1.1 |
| **에너지응축** | 5/20/50MA가 촘촘히 모인 삼각수렴 끝단 | MA 간격 좁음 + 20일 레인지 ≤14% |

> 한 종목이 여러 유형에 해당하면 **눌림목 → 돌파 → 에너지응축** 순으로 하나만 표시합니다.

## 2. 나스닥 설정값

### 기본 필터 (1단계)

| 설정 | 기본값 | 의미 |
|------|:------:|------|
| `ND_MIN_CAP_B` | **10** | 시총 하한 ($B) |
| `ND_MAX_DEBT`  | **200** | 부채비율 상한 (%) — 데이터 없으면 통과 |

### 펀더멘털 필터 (성장주 — 1단계-b)

PER 절대값보다 **ROE·매출성장·PEG**를 우선하고, PER이 과도하면 제외합니다.

| 설정 | 기본값 | 의미 |
|------|:------:|------|
| `ND_ROE_MIN` | **0.07** | ROE 하한 (7%) — **미확인 시 탈락** |
| `ND_REV_GROWTH_MIN` | **0.05** | 매출성장 하한 (5%) |
| `ND_PEG_MAX` | **2.5** | PEG 상한 — **forward PER 우선**, 없으면 trailing |
| `ND_PER_MAX` | **80** | **trailing PER** 절대 상한 — 과열 종목 제외 |

> PEG = PER ÷ 매출성장(%). 지속 상승·성장주는 trailing PER만 보면 과대평가로 보일 수 있어 **forward PEG를 먼저** 씁니다. PER 80 상한은 **trailing** 기준(현재 이익 대비 과열 방어).

> 영업적자(`profit_margin ≤ 0`)도 탈락. 기술 조건(눌림목/돌파/압축)을 통과해도 펀더 미달이면 `entry_type` 없음 → **「기술 통과·펀더 미달」** 섹션에 표시.

### 눌림목 기준

| 설정 | 기본값 | 의미 |
|------|:------:|------|
| `PB_5MA_RANGE`   | **±2.5%** | 5MA 이격 허용 범위 (프롬프트 10일선 → 5MA 근사) |
| `PB_FROM_20MA`   | **-4.0%** | 5-20MA이격이 이보다 아래면 제외 (너무 깊은 이탈) |
| `ND_EXT_20MA_MAX`| **8.0%** | 현재가 20MA 대비 이격 상한 (§6 과열 제외) |
| `PB_RSI_MIN/MAX` | **38~65** | RSI 범위 |
| `PB_VOL_MAX`     | **1.05x** | 5d/20d 거래량 ≤ 1.05 (수축) |

> 눌림목은 **5>20>50 정배열** + **RS▲** 필수. 거래량 수축이 정석적인 눌림목.

### 돌파 기준

| 설정 | 기본값 | 의미 |
|------|:------:|------|
| `BK_FROM_HIGH`   | **-6.0%** | 52주 고점 대비 최소 거리 |
| `BK_RSI_MIN/MAX` | **58~82** | RSI 범위 |
| `BK_VOL_MIN`     | **1.1x** | 5d/20d 거래량 ≥ 1.1 (급증) |

> 돌파는 반드시 **200MA 위 + 20MA 위** 종목만 해당.

### 에너지응축 기준

| 설정 | 기본값 | 의미 |
|------|:------:|------|
| `SQ_5_20_MAX`    | **5.0%** | \|5MA − 20MA\| / 20MA |
| `SQ_20_50_MAX`   | **8.0%** | \|20MA − 50MA\| / 50MA |
| `SQ_RANGE_MAX`   | **14.0%** | 20일 (최고−최저) / 현재가 |
| `SQ_RSI_MIN/MAX` | **43~70** | RSI 범위 |

### 시장 레짐 (QQQ)

- **통과**: QQQ > 50MA AND 50MA 우상향 → 🟢 진입 허용
- **실패**: 🔴 진입 보류 **경고** — 후보는 계속 표시. (진입 유형 분류 자체를 막지는 않음)

### 장중 거래량 보정

| 설정 | 기본값 | 의미 |
|------|:------:|------|
| `INCOMPLETE_VOL_RATIO` | **0.50** | 당일 거래량 < 20일평균×50% → 미완성 봉으로 보고 **전일** 기준 사용 |

출력에 `[장중추정]`이 붙으면 이 보정가 적용된 것입니다.

## 3. 나스닥 출력 구성

1. **레짐 헤더** — QQQ 현재가, 50MA, 20일 수익률
2. **눌림목 후보** — 5MA 근접 + RSI 건강 + 거래량 수축 (RS▲/▼ 표시)
3. **돌파 후보** — 52주 고점 근접 + 거래량 급증
4. **에너지응축 후보** — MA 간격 좁은 압축 구간
5. **QQQ 상대강도 Top 10** — 20일 수익률 기준
6. **전체 요약** — RSI·5MA이격·고점대비·RS·진입유형/제외태그

### 출력 태그

| 태그 | 의미 |
|------|------|
| `[눌림목]` / `[돌파]` / `[에너지응축]` | 해당 진입 유형 통과 |
| `RS▲` / `RS▼` | QQQ 대비 20일 수익률 우위/열위 |
| `추세X` | 주가 < 50MA 또는 50MA 비우상향 |
| `시총↓` | 시총 $10B 미만 |
| `부채↑` | 부채비율 200% 초과 |
| `ROE?` / `ROE↓` | ROE 미확인 / 7% 미만 |
| `성장?` / `성장↓` | 매출성장 미확인 / 5% 미만 |
| `PEG↑` | forward PEG(없으면 trailing) 2.5 초과 |
| `PER↑` / `PER?` | **trailing** PER 80 초과 / 적자·미확인 |
| `적자` | 영업이익률 ≤ 0 |
| `RSI↓` / `RSI↑` | RSI 범위 미달/초과 |
| `고점↓` | 52주 고점 대비 과도한 하락 |
| `레짐↓` | 시장 레짐 약세 |
| `관찰` | 추세 OK, 진입 유형 해당없음 |
| `[장중추정]` | 전일 거래량 기준으로 보정 |

## 4. 코스피 모드 — 개요

`코스피_후보발굴_프롬프트.md` 기준을 **네이버 금융** 펀더 + Yahoo 차트 데이터로 **10점 점수화**.

### 흐름도

```
코스피 후보 종목 (~120개)
      │
  ① 코스피 레짐 (KOSPI > 50MA + 우상향)   → 약세면 경고
      │
  ② 10점 만점 점수 계산
      ├─ 가치/안전마진  (3점): PBR, PER
      ├─ 실적/성장     (4점): 매출성장, 이익률, 목표주가 괴리율
      └─ 수급/기술     (3점): 20MA 위, RSI, 기관+외인 5일 순매수
      │
  ③ 필수 탈락 기준 (부채비율, ROE, PBR, PER 극단값)
      │
  ④ 분류
      ├─ 우선 후보  : 7점 이상
      ├─ 관찰 후보  : 5~6점
      └─ 조건 미달  : 4점 이하 또는 탈락 기준 해당
```

### 코스피 점수표 (10점 만점)

| 항목 | 기준 | 점수 |
|------|------|:----:|
| 가치: PBR | ≤ 1.0 (`KS_PBR_GOOD`) | +2 |
| 가치: PBR | 1.0 ~ 1.5 | +1 |
| 가치: PER | ≤ 12 (`KS_PER_GOOD`) | +1 |
| 실적: 매출성장 | > 0% | +1 |
| 실적: 영업이익률 | > 0% | +1 |
| 실적: 목표주가 괴리율 | ≥ 20% (`KS_TGT_GAP`) | +2 |
| 기술: 20MA 위 | 현재가 > 20MA | +1 |
| 기술: RSI | 38 ~ 70 | +1 |
| 수급: 기관+외국인 | 5일 누적 순매수 > 0 **또는** 10·20일 동시 순매수 > 0 (네이버) | +1 |

### 코스피 설정값

| 설정 | 기본값 | 의미 |
|------|:------:|------|
| `KS_PBR_MAX` | **1.5** | PBR 상한 (안전마진) |
| `KS_PBR_GOOD` | **1.0** | PBR 우수 (+2점) |
| `KS_PER_MAX` | **22** | PER 상한 (극단 고평가 제외) |
| `KS_PER_GOOD` | **12** | PER 우수 (+1점) |
| `KS_ROE_MIN` | **7%** | ROE 하한 |
| `KS_DEBT_MAX` | **200%** | 부채비율 상한 |
| `KS_RSI_MIN/MAX` | **38~70** | RSI 범위 |
| `KS_FLOW_DAYS` | **5** | 기관+외인 누적 순매수 가점 일수 |
| `KS_TGT_GAP` | **20%** | 목표주가 괴리율 하한 (프롬프트 §2·§6·§9) |

### 필수 탈락 기준

아래 **하나라도** 해당하면 점수와 무관하게 **조건 미달**:

| 기준 | 탈락 조건 |
|------|-----------|
| 부채비율 | D/E > 200% (**금융주 500%↑ 면제**) |
| ROE | ROE < 7% (데이터 있을 때만) |
| PBR | PBR > 1.5 |
| PER | PER > 22 또는 PER < 0 (적자) |

```
1) 종목 리스트 고정 — 리스트 밖은 스캔 불가 (나스닥 ~220 / 코스피 ~120)
2) 코스피 수급은 네이버 금융 일별 기관·외국인 순매매량 (Yahoo 미제공)
3) 코스피 어닝 서프라이즈·컨센서스 상향은 목표가 괴리율로 간접 측정
4) 재무 데이터는 분기 시차 가능
5) 장중 실행 시 당일 거래량·수급은 미확정일 수 있음
6) 일부 티커는 Yahoo 오류로 스킵될 수 있음 (로그에만 표시)
7) 스크리너는 후보 압축 도구 — 최종 매수는 뉴스·실적·손익비 별도 확인
```
