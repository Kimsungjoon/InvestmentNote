# 매수 후보 일일 워크플로

> **매일 이 문서만 보면 됩니다.**  
> 원칙: **스크리너(숫자) → 발굴 프롬프트(정성 보완) → 최종분류(매수 검토)**  
> 스크리너와 프롬프트는 **같은 일을 두 번** 하는 게 아니라, **역할이 다릅니다.**

**한 방에 끝내기:** 채팅에서 `/나스닥_매수후보_스크린` 또는 `/코스피_매수후보_스크린` → ① 스크리너 + ② 통과(우선·관찰) 종목 후보발굴 + ③ 요약까지 **자동** (규칙: `.cursor/rules/buy-candidate-post-screen.mdc`)

## 🇺🇸 나스닥 (추세추종 · 2주~2개월)

### 체크리스트

- [ ] **①~③** `/나스닥_매수후보_스크린` 실행 → 스크리너 + **통과 종목 후보발굴(§7)** + 요약까지 자동
- [ ] (수동 ②만 할 때) 리포트 `@첨부` + 아래 「짧은 예시」 또는 「2차 프롬프트」
- [ ] 당일 리포트 확인: `b_매수후보리스트/나스닥/나스닥_매수후보_YYYY-MM-DD.md`
- [ ] **레짐** — QQQ 50MA 위/아래 → 실탄 매수 보류 여부 확인
- [ ] **③** 최종분류·1순위·보유(HWM 등) 확인
- [ ] **보유 종목 있으면:** 리포트 손절·목표가가 `보유주식/나스닥.md`와 같은지 확인 (다르면 **파일 기준**)

### ① 스크리너가 하는 일 (다시 계산하지 말 것)

- QQQ RS · 눌림목/돌파/압축 · 정배열 · 펀더(PEG·ROE·성장)
- RR · 4그룹 분류 · 레짐 경고

### ② AI가 보완할 것 (발굴 프롬프트)

| 섹션 | 내용 |
|------|------|
| §1 | 섹터 ETF·자금 흐름 |
| §6 | 급등 후 거래량 급감, 테마-only |
| §7 | 통과 종목 통합 재정리 (선택) |

### ② 채팅 프롬프트 예시 (짧게)

리포트·후보발굴 프롬프트를 `@`로 첨부한 뒤, 아래 중 하나를 붙여넣으면 됩니다.

```
@b_매수후보리스트/나스닥/나스닥_매수후보_YYYY-MM-DD.md
@a_프롬프트/나스닥/나스닥_후보발굴_프롬프트.md

아래 리포트 통과 종목만 대상으로 §7 기준으로 재검토해줘.
§1 섹터·§6 제외는 AI로 보완.
```

```
@b_매수후보리스트/나스닥/나스닥_매수후보_YYYY-MM-DD.md
@a_프롬프트/나스닥/나스닥_후보발굴_프롬프트.md

통과 종목만 §7로 우선/관찰/제외 정리해줘.
섹터 ETF(§1)·급등 리스크(§6) 빈칸 채워서 리포트에 반영.
스크리너 숫자는 재계산하지 마.
```

### 2차 프롬프트 (상세 · 복사용)

```
@b_매수후보리스트/나스닥/나스닥_매수후보_YYYY-MM-DD.md
@a_프롬프트/나스닥/나스닥_후보발굴_프롬프트.md

위 리포트 **통과 종목만** 대상으로:
1. §1 — 섹터·자금 흐름 관점 보완
2. §6 — 급등 후 거래량 급감·테마-only 해당 여부
3. §7 — 우선/관찰/제외 한 줄 재정리

규칙:
- 스크리너 숫자(PER·PEG·RS·RR)는 리포트 기준 유지, 재계산 금지
- 보완은 당일 리포트 파일에만 반영 (새 파일 만들지 말 것)
- QQQ 레짐 약세면 실탄 매수 보류 명시
```

@a_프롬프트/코스피/코스피_후보발굴_프롬프트.md

2. §2·§3 — 실적·키맞추기 (해당 시)
3. §7 — 오버행·거래대금·공시 리스크
4. §9 — 핵심 근거 1줄 + 최대 리스크 1줄

규칙:
- PBR/PER/점수/수급은 리포트·네이버 기준 유지

- 코스피는 **자산 방어** — 불확실하면 관망

## 하지 않을 것

| ❌ | 이유 |
|----|------|
| 스크리너 없이 발굴 프롬프트만 | 기준 들쭉날쭉 |

| §1~§9 전부 매번 | §7·§9 + 필요한 §만 |
| 날짜 다른 리포트 새로 만들기 | `YYYY-MM-DD` 당일 파일만 갱신 |
