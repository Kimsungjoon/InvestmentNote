# 코스피 매수후보 스크린

코스피 **가치투자·자산 방어** 전략에 맞춰 **① 스크리너 → ② 후보발굴(우선·관찰만) → ③ 최종분류**까지 한 번에 수행한다.

**② 기준:** `매수후보_일일워크플로!!!!!!.md` — 코스피 「2차 프롬프트 (상세 · 복사용)」  
**규칙:** `.cursor/rules/buy-candidate-post-screen.mdc`

## 즉시 실행 (순서 고정)

### ① 스크리너 + 리포트

1. 아래 명령을 실행하고 완료까지 대기한다 (네트워크 필요, 약 1~3분).

```bash
python3 screener.py --report-kospi
```

2. 생성 파일 확인 — `b_매수후보리스트/코스피/코스피_매수후보_YYYY-MM-DD.md` 를 읽는다.

### ② 후보발굴 (우선·관찰만 — ① 직후 같은 턴에서 실행)

3. 아래 파일을 읽고 **우선·관찰 종목만** 대상으로 후보발굴을 수행한다.
   - `a_프롬프트/코스피/코스피_후보발굴_프롬프트.md`
   - `매수후보_일일워크플로!!!!!!.md` (2차 프롬프트 문구 준수)

4. **§1** 섹터 공포·저평가 · **§2·§3** 실적·키맞추기 · **§7** 오버행·공시 · **§9** 핵심 근거·리스크 1줄  
   - PBR·점수·수급은 **재계산 금지** — 리포트·네이버 기준 유지  
   - 보완은 **당일 리포트 파일만** 수정

### ③ 교차 검증 + 요약

5. 아래 파일과 대조한다.
   - `보유주식/코스피.md` (있을 경우 — 보유·손절·목표가)
   - `a_프롬프트/코스피/코스피_최종분류_프롬프트.md` (4그룹 분류)
   - `.cursor/rules/trading-profile.mdc` (코스피 가치·방어 전략)

6. **한국어 요약 보고** — 다음을 포함한다.
   - KOSPI 레짐 (50MA 대비)
   - 우선/관찰/미달/제외 종목 수와 핵심 티커
   - **신규 1순위** · **보유 재검토** (있을 때)
   - 자산 방어 관점 매수 가능 여부
   - 리포트 파일 경로

## 전략·기준

- **가치투자 + 자산 방어** — 고변동·추격 매수 배제
- 스크리너 10점: 가치(3) + 실적/성장(4) + 수급/기술(3)
- **펀더 데이터: 네이버 금융** (PER/PBR/ROE/부채비율/목표주가·재무) — Yahoo는 누락 시만 보완
- 우선 후보: 점수 7+ · 관찰: 5~6 · 필수 재무 미달 → 제외
- 최종 **우선 검토**: 점수 7+ + 기관·외인 5일 순매수
- 손절: min(50MA, 20일 스윙저점) × 0.98 · 목표: 애널 mean

## 금지

- 스크리너 없이 리포트만 작성
- 당일 파일이 있는데 새 파일 중복 생성
