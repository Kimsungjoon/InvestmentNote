# 나스닥 매수후보 스크린

나스닥 추세추종 스윙 전략에 맞춰 **① 스크리너 → ② 후보발굴(통과 종목만) → ③ 최종분류**까지 한 번에 수행한다.

**② 기준:** `매수후보_일일워크플로!!!!!!.md` — 나스닥 「2차 프롬프트 (상세 · 복사용)」  
**규칙:** `.cursor/rules/buy-candidate-post-screen.mdc`

## 즉시 실행 (순서 고정)

### ① 스크리너 + 리포트

1. 아래 명령을 실행하고 완료까지 대기한다 (네트워크 필요, 약 15~30초).

```bash
python3 screener.py --report
```

2. 생성 파일 확인 — `b_매수후보리스트/나스닥/나스닥_매수후보_YYYY-MM-DD.md` 를 읽는다.

### ② 후보발굴 (통과 종목만 — ① 직후 같은 턴에서 실행)

3. 아래 파일을 읽고 **리포트 통과 종목만** 대상으로 후보발굴을 수행한다.
   - `a_프롬프트/나스닥/나스닥_후보발굴_프롬프트.md`
   - `매수후보_일일워크플로!!!!!!.md` (2차 프롬프트 문구 준수)

4. **§1** 섹터·자금 흐름 · **§6** 급등 후 거래량 급감·테마-only · **§7** 우선/관찰/제외 재정리  
   - 스크리너 숫자(PER·PEG·RS·RR)는 **재계산 금지** — 리포트 기준 유지  
   - 보완 내용은 **당일 리포트 파일만** 수정 (새 파일 금지)

### ③ 교차 검증 + 요약

5. 아래 파일과 대조한다.
   - `보유주식/나스닥.md` (보유·손절·목표가)
   - `a_프롬프트/나스닥/나스닥_최종분류_프롬프트.md` (4그룹 분류)
   - `.cursor/rules/trading-profile.mdc` (나스닥 전략)

6. **한국어 요약 보고** — 다음을 포함한다.
   - QQQ 레짐 (50MA 대비, 진입 허용/보류)
   - 스크리너 통과 N종목 · 우선/관찰/미달/제외 각 몇 종목
   - **신규 1순위** · **보유 재검토** 종목 (있을 때)
   - 레짐 기준 **실탄 매수 가능 여부**
   - 리포트 파일 경로

## 전략·기준

- 보유 목표: 2주~2개월 스윙. QQQ 50MA 미회복 시 **실탄 매수 보류**.
- 손절: min(50MA, 20일 스윙저점) × 0.98
- 목표: Yahoo 애널 mean (`targetMeanPrice`)
- RR: (목표 − 현재) ÷ (현재 − 손절)
- 펀더: forward PEG 우선 (`screener.py`와 동일)
- 우선 검토: 정배열 + RS▲ + 거래량 + RR≥2 + 레짐 🟢

## 금지

- 스크리너 없이 리포트만 작성
- 당일 파일이 있는데 새 파일 중복 생성
