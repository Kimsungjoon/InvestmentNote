# 나스닥 매수후보 스크린 워크플로우

> **트리거 (권장):** Agent 채팅에서 `/` 입력 → **`나스닥_매수후보_스크린`** 선택  
> 명령 파일: `.cursor/commands/나스닥_매수후보_스크린.md`

---

## 목적

1. `screener.py`로 나스닥 후보 스캔
2. RR·최종분류·선정/탈락 이유를 담은 리포트를 `b_매수후보리스트/나스닥/`에 저장
3. 에이전트가 리포트를 검토하고 필요 시 보완 의견 추가

---

## 실행 순서

### 1단계 — 스크리너 + 리포트 생성

```bash
python3 screener.py --report
```

- 약 2~3분 소요 (Yahoo API 병렬 조회)
- 출력 파일: `b_매수후보리스트/나스닥/나스닥_매수후보_YYYY-MM-DD.md`

### 2단계 — 컨텍스트 확인

아래 파일을 읽고 리포트와 교차 검증:

| 파일 | 용도 |
|------|------|
| `보유주식/나스닥.md` | 보유 종목·손절/목표가 |
| `a_프롬프트/나스닥/나스닥_최종분류_프롬프트.md` | 4그룹 분류 기준 |
| `.cursor/rules/trading-profile.mdc` | 추세추종·레짐·RR 전략 |

### 3단계 — 리포트 검토·보완

자동 생성 리포트를 기준으로:

1. **레짐 경고**가 맞는지 확인 (QQQ vs 50MA)
2. **제외/조건 미달** 종목의 애널 목표가·RR이 타당한지 확인
3. **관찰/우선 검토** 종목에 시장·업종 맥락 1~2줄 보완 (필요 시)
4. 보유 종목은 `보유주식/나스닥.md` 손절·목표가와 불일치 없는지 확인

보완이 필요하면 **같은 날짜 파일**을 직접 수정한다. 새 파일을 만들지 않는다.

### 4단계 — 사용자에게 요약 전달

다음 형식으로 짧게 보고:

- 시장 레짐 (진입 허용/보류)
- 스크리너 통과 N종목 · 우선/관찰/미달/제외 각 몇 종목
- **신규 1순위** · **보유 재검토** 종목 (있을 때)
- 실탄 매수 가능 여부 (레짐 기준)
- 리포트 파일 경로

---

## 산출물 기준

| 항목 | 기준 |
|------|------|
| 손절가 | min(50MA, 20일 스윙저점) × 0.98 |
| 목표가 | Yahoo `targetMeanPrice` (애널 mean) |
| RR | (목표 − 현재) ÷ (현재 − 손절) |
| 펀더 PEG | forward PEG 우선 (`screener.py`와 동일) |
| 우선 검토 | 정배열 + RS▲ + 거래량 + RR≥2 + 레짐 🟢 |
| 제외 | 목표가 < 현재가, 추세 훼손, RSI 과열 |
