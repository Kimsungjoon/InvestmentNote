# 코스피 매수후보 스크린 워크플로우

> **트리거 (권장):** Agent 채팅에서 `/` 입력 → **`코스피_매수후보_스크린`** 선택  
> 명령 파일: `.cursor/commands/코스피_매수후보_스크린.md`

---

## 목적

1. `screener.py --report-kospi`로 코스피 후보 스캔
2. RR·최종분류·선정/탈락 이유를 담은 리포트를 `b_매수후보리스트/코스피/`에 저장
3. 에이전트가 리포트를 검토하고 필요 시 보완

---

## 실행 순서

### 1단계 — 스크리너 + 리포트 생성

```bash
python3 screener.py --report-kospi
```

- 출력: `b_매수후보리스트/코스피/코스피_매수후보_YYYY-MM-DD.md`

### 2단계 — 컨텍스트 확인

| 파일 | 용도 |
|------|------|
| `보유주식/코스피.md` | 보유 종목·손절/목표가 (있을 때) |
| `a_프롬프트/코스피/코스피_최종분류_프롬프트.md` | 4그룹 분류 |
| `.cursor/rules/trading-profile.mdc` | 코스피 가치·방어 전략 |

### 3단계 — 리포트 검토·보완

1. **재무 제외** (부채·ROE·PBR·PER) 타당성
2. **수급** (네이버 5일 기관·외인 순매수) 방향
3. **목표괴리·RR** — 보수적 방어선과 충돌 없는지
4. 보유 종목은 `보유주식/코스피.md`와 일치 확인

### 4단계 — 사용자 요약

- KOSPI 레짐
- 우선/관찰/미달/제외 종목 수
- 1순위 후보 + 자산 방어 관점 매수 가능 여부
- 리포트 경로

---

## 산출물 기준

| 항목 | 기준 |
|------|------|
| **데이터 소스** | 네이버 금융 (PER/PBR/ROE/부채·목표·수급) · Yahoo는 차트·보완 |
| 우선 후보 | 점수 ≥ 7 |
| 관찰 후보 | 점수 5~6 |
| 제외 | ROE·부채·PBR·PER 필수 미달 |
| 최종 우선 검토 | 점수7+ + 수급▲ |
| 손절 | min(50MA, 20일 스윙저점) × 0.98 |
| 목표 | 네이버 컨센서스 목표주가 |
